>_ELISPRS REFERENCE

An Emacs Lisp runtime in Rust that runs .el outside Emacs. A Lisp-2 obarray (separate value/function cells) with dynamic binding and an elisp-correct reader, built on the rust_lisp value model and being lowered onto the fusevm bytecode VM — the same engine behind zshrs, stryke, awkrs, and vimlrs. Milestone 1 / early.

What it is

Emacs Lisp has only ever run inside Emacs. elisprs takes the language out of the editor and runs .el as ordinary programs from the command line. It is built to become a fusevm frontend — the fifth language on the shared VM — so that elisp eventually executes on the same Cranelift-JIT/AOT engine as the sibling languages rather than a bespoke interpreter.

Today (milestone 1) elisp runs on a self-contained tree-walk evaluator. The fusevm lowering (src/compiler.rs) is the milestone-2 seam and is not wired yet — so, unlike vimlrs, elisprs is being lowered onto fusevm, not yet hosted on it.

Architecture

Emacs Lisp is a Lisp-2 (separate value and function cells per symbol) and dynamically scoped — exactly what an off-the-shelf Lisp won't give you. So elisprs splits the problem: reuse a value model, own the semantics.

.el source  →  reader.rs  →  Value (rust_lisp model)  →  eval (Lisp-2 + dynamic)      [milestone 1]
                                                      ↘  compiler.rs → fusevm::Chunk      [milestone 2]

Data model reused

The Value / List / Symbol types come from rust_lisp (MIT) — the well-trodden value representation.

Reader is ours

rust_lisp's parser mis-tokenizes core elisp syntax (1+/1-, #'foo, ?c, dotted pairs), so the reader is a small elisp-correct one that emits rust_lisp Values.

Lisp-2 + dynamic

A custom obarray gives every symbol a value cell and a function cell; let binds dynamically via a specstack. rust_lisp's Lisp-1/lexical eval is not used.

Functions in Foreign

Closures, macros, and subrs ride in Value::Foreign(Rc<dyn Any>) and downcast on the way out — elisp function semantics without forking the value enum.

A taste

(defun fact (n) (if (<= n 1) 1 (* n (fact (1- n)))))
(fact 6)                                  ; => 720

(mapcar (lambda (x) (* x x)) '(1 2 3 4))  ; => (1 4 9 16)
(mapcar #'1+ '(10 20 30))                 ; => (11 21 31)

(let ((x 10) (y 20)) (+ x y))             ; => 30

(format "%s = %d (hex %x)" 'count 255 255); => "count = 255 (hex ff)"

(condition-case e (/ 1 0)
  (arith-error (format "caught %s" e)))   ; => "caught (arith-error division by zero)"

Why elisprs

Elisp, standalone

Run .el as a program from the shell, with a REPL — no Emacs process required.

Faithful semantics

A real Lisp-2 with dynamic binding and separate function/value cells, not a Scheme wearing an Emacs hat.

One shared engine (soon)

Once lowered, elisp inherits fusevm's Cranelift JIT and AOT — the same speedups that land for zshrs, stryke, awkrs, and vimlrs.

Honest about gaps

Unsupported syntax (dotted pairs, backquote) errors loudly rather than silently misreading your config.

Building from source

elisprs builds as a standalone Rust crate. Milestone 1 has a single dependency (rust_lisp) and builds offline:

git clone https://github.com/MenkeTechnologies/elisprs
cd elisprs

cargo build --release
cargo test            # reader unit tests + end-to-end eval suite

./target/release/elisp examples/demo.el
